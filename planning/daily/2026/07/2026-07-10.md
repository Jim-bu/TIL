## 🔵 Tech → Product (기술에서 서비스로)

**pentagi·strix(자율 침투테스트 에이전트) → Perfai Security(1-프롬프트 취약점 수정)**

`pentagi`(Go)와 `strix`는 사람 대신 취약점을 탐지·재현·검증하는 **자율 보안 에이전트**다. 핵심은 에이전트가 스캔→익스플로잇 시도→결과 검증을 반복하되, 위험한 액션은 격리·승인 흐름 안에서 돌리는 태스크 플래닝.

이 기술을 소비자용으로 포장하면 **Perfai Security**가 된다 — "Vibe App(바이브 코딩으로 급조한 앱)의 라이브 취약점을 **1-프롬프트로 찾고 고친다**". 즉 *자율 스캔 → 코드 컨텍스트 기반 취약점 판정 → 패치 생성 → 재검증* 파이프라인을, 보안 전문성이 없는 개발자도 프롬프트 한 줄로 쓰게 만든 것. 오픈소스 보안 에이전트가 "전문가 도구 → 누구나 쓰는 버튼"으로 전환된 사례.

## 🟢 Product → Tech (서비스에서 기술로)

**Toyo(iMessage 실행비서) → DesktopCommanderMCP·MCP 도구 연동**

**Toyo**는 "iMessage에 살면서 전화까지 거는 실행비서"다. 역으로 뜯으면: 사용자가 메시지로 지시 → 에이전트가 **여러 외부 도구(캘린더·이메일·PM)를 실제로 조작** → 결과를 대화로 회신. 관건은 "말"을 "실제 도구 실행"으로 안전하게 잇는 **도구 연동 계층**이다.

그 계층의 표준 구현이 GitHub의 **DesktopCommanderMCP** 같은 **MCP 서버**다 — 터미널·파일시스템 같은 실제 능력을 에이전트에 노출하되, 권한·안전장치를 두는 구조. Toyo가 메시징 채널에서 하는 일을, MCP 서버는 로컬 자원에서 한다. MCP 서버가 툴을 어떻게 노출하고 위험 명령을 어떻게 가두는지 보면, Toyo류 "행동하는 비서"의 하부가 그려진다.

## 💡 Deep Dive — 오늘의 인사이트

- **🧠 주요 개념** — *Agent Hands (도구 실행 레이어).* 어제의 "게이트웨이(두뇌로 가는 길)"에 이어, 오늘은 에이전트가 실제로 **OS·오피스·보안 도구를 조작하는 손**(DesktopCommanderMCP·OfficeCLI·pentagi)이 핵심. MCP가 그 표준 인터페이스로 자리잡는 중.
- **📦 주목 프로덕트** — *Perfai Security.* "1-프롬프트로 취약점 탐지·수정"으로 **바이브 코딩의 부작용(급조 앱의 보안 구멍)** 이라는 신생 페인포인트를 정확히 겨냥. 기술 유행이 만든 새 문제를 곧바로 제품화.
- **🎯 핵심 기획력** — *사용자가 이미 있는 곳으로 간다.* Toyo는 새 앱 설치를 요구하지 않고 **iMessage 안**으로 들어갔다. "새 습관을 만들지 말고 기존 습관에 올라타라"는 배포 전략의 정석.
- **한 줄 종합:** 에이전트 경쟁의 무게중심이 *"무엇을 아는가(모델·게이트웨이)"* 에서 *"무엇을 실제로 할 수 있는가(도구 실행·MCP)"* 로 이동하고 있다.
