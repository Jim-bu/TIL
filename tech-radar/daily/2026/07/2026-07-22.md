## 🔵 Tech → Product (기술에서 서비스로)

**1jehuang/jcode(에이전트 하네스) → Rerun(누구나 쓰는 에이전트 빌더)**

`jcode`는 "에이전트가 코드를 안정적으로 실행/편집"하게 하는 **하네스** — 컨텍스트 관리, 툴 실행 루프, 결과 검증·재시도 같은 저수준 배관이다. 에이전트가 헛짓 없이 일하려면 이 하네스 품질이 핵심.

이 하네스 아이디어를 **노코드 제품**으로 올리면 **Rerun**("모든 업무용 에이전트를 가장 쉽게 만든다")이 된다. 즉 *툴 커넥터 + 오케스트레이션 + 검증 루프*를 UI로 감싼 것. jcode류의 하네스 코드를 보면, Rerun 같은 빌더가 내부적으로 무엇(컨텍스트·툴콜·재시도)을 자동화해 주는지가 그려진다. **하네스(런타임) → 빌더(제품)** 의 전형.

**CartAI**는 "결제/체크아웃을 대신 처리하는 에이전트"다. 역으로 뜯으면: ① 다양한 쇼핑몰의 체크아웃 폼을 다루는 **브라우저 자동화(Playwright류)**, ② 결제수단·주소 등 **상태·시크릿 관리**, ③ 폼 변형·에러에 견디는 **복원력**, ④ 무엇을 얼마까지 결제할지의 **지출 가드레일**.

④가 진짜 급소인데, 이는 07-15의 **Agentcard(에이전트 데빗카드)** 와 07-13의 **destructive_command_guard(위험 액션 차단)** 와 동일한 "자율 행동에 정책 브레이크" 패턴이다. 즉 CartAI가 성립하려면 *웹 자동화 + 결제 레일 + 실시간 승인 정책*이 맞물려야 한다. GitHub의 에이전트 하네스(jcode)가 "행동의 신뢰성"을, 가드레일이 "행동의 안전"을 담당한다.

## 🔬 Enterprise Deep Dive — AI/ML 공급망 보안 in CI/CD (KodeKloud 발)

> 오늘의 1 Pick. 에이전트가 모델·의존성을 자동으로 끌어다 쓰기 시작하면서, "무엇을 파이프라인에 들이나"가 새 공격면이 됨.

### 왜 쓰나 (도입 근거)
- **해결 문제**: 모델 가중치·데이터셋·ML 라이브러리가 이제 **공급망 아티팩트**. 위험 — ① `pickle` 역직렬화로 **임의 코드 실행**, ② 포이즌된 가중치/백도어, ③ 취약·오염된 의존성, ④ 출처 불명 서드파티 모델. 에이전트가 자동 fetch하면 공격면 급증.
- **대안 대비**: 전통 SCA(코드 의존성 스캔)는 **모델 아티팩트를 못 본다** → ML 전용 스캐닝·서명이 필요.
- **도입 효과**: 오염 모델·악성 직렬화 차단, 출처(provenance) 보장, 규제/감사 대응.
- **비용/리스크**: 대용량 모델 스캔 시간, 서명키 관리, 서드파티 허브(HF) 신뢰 경계.

### 어떻게 연동하나 (기술 통합)
- **스택 위치**: CI/CD 파이프라인에 **게이트** 삽입 — 모델·의존성이 레지스트리/배포로 가기 전에 스캔·서명 검증.
- **최소 통합(개념 예시)**:
```yaml
# CI 스텝 예시 (게이트: 통과 못 하면 파이프라인 실패)
- run: modelscan -p ./artifacts/model.pkl      # 위험 opcode/역직렬화 탐지
- run: syft ./artifacts -o spdx-json > sbom.json # ML SBOM(의존성·출처)
- run: cosign verify --key $KEY ghcr.io/org/model:tag  # 서명·프로버넌스 검증
```
- **운영 고려**: 가능하면 `pickle` 대신 **safetensors** 포맷 채택(역직렬화 실행 위험 제거). 배포 단엔 **admission control**(서명·스캔 통과만 허용). 모델 서명은 Sigstore/cosign로.
- **함정(gotchas)**: ① 레거시 pickle 모델을 safetensors로 못 바꾸는 경우, ② 대형 모델 스캔 지연, ③ 서드파티 허브 모델의 출처 불명, ④ 서명키 유출 시 전체 신뢰 붕괴.

### 도입 판단 체크리스트
- [ ] CI/CD가 **외부/서드파티 모델**을 끌어오는가?
- [ ] `pickle` 등 **역직렬화형 포맷**을 쓰는가?
- [ ] **에이전트가 모델·의존성을 자동 fetch**하는가?
- [ ] 규제·감사 대상 산업인가?

→ 2개 이상 Yes면 CI에 **modelscan + SBOM + 서명검증** 게이트부터 넣는다. (safetensors 전환은 가장 싸고 효과 큰 첫 수.)
