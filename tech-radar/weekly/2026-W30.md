# 2026-W30 Weekly — PM Digest (2026-07-20 ~ 07-26)

> AI 솔루션 딜리버리 PM 관점 정리 — 계약·제안 → 수행/협업 → 검수·테스트 → **Docker 배포·설치** → 고객 인프라/서비스 안내.

**이번 주 한 문장:** 업계의 무게중심이 *"에이전트가 뭘 하나(모델·기능)"* 에서 **"프로덕션 조직에 어떻게 안전하게 얹고(거버넌스·격리) 얼마에 운영하나(자체호스팅·GPU 원가)"** 로 확실히 이동했다 — 정확히 딜리버리 PM의 영역.

## 🧭 PM 업무 단계별 시사점

### 1) 계약·제안 (Pre-sales / Scoping)
- **배포 모델을 계약서에 명시**: 온프렘(고객 인프라 내 Docker) vs 클라우드/매니지드. 데이터 주권 요구가 커져(29→54%), 온프렘·자체 추론이 강력한 세일즈 포인트.
- **거버넌스·감사 요구 선반영**: 고객이 "데이터 어디 가나 / 감사되나 / 권한 통제되나"를 물을 것 → 제안서에 감사로그·최소권한·데이터 사내보관 항목.
- **GPU 원가 근거 확보**: 자체호스팅의 최대 비용은 GPU인데 흔히 60% 유휴 → 과다/과소 견적 방지 위해 사이징 근거(활용률·공유) 준비.

### 2) 수행·협업 (개발자·디자이너 소통)
- **개발자와 공유할 개념**: MCP(우리 솔루션↔고객 시스템 연동 표준), 에이전트 메모리/컨텍스트(cognee류), 게이트웨이(OmniRoute류 멀티모델 라우팅).
- **디자이너와**: 인앱 에이전트(Skippr류)·사람+에이전트 병렬 UX(ego-lite) — "AI가 제품 안에서 어떻게 보이고 개입하나".
- **소통 원칙**: "모델 성능"보다 **신뢰성·일관성·실패 시 동작**을 스펙으로 합의(업계도 그쪽으로 이동).

### 3) 검수·테스트 (QA / 프로젝트 종료)
- **검수 기준에 신뢰성 항목 추가**: 출력 일관성, 실패 시 롤백/폴백, 전 액션 감사로그. (업계 화두 = "reliability, not capability")
- **배포 전 시뮬레이션**: 대형 벤더도 배포 전 에이전트 스트레스 테스트(시뮬 환경)를 넣음 → 검수 단계에 유사 시나리오 테스트 도입 고려.
- **검색 품질(retrieval) 점검**: 에이전트 답 품질이 곧 컨텍스트/검색 품질 → 검수 시 "엉뚱한 답의 원인이 데이터인지 모델인지" 구분.

### 4) 배포·설치 (Docker 이미지 / 온프렘) — ★ PM 핵심
- **ML 공급망 보안**: 이미지에 모델 포함 시 — 가능하면 `pickle` 대신 **safetensors**, 이미지·모델 **서명(cosign)**, 취약점 **스캔(modelscan)**, **SBOM** 동봉. 고객 보안팀 심사 통과의 열쇠.
- **격리·최소권한**: 우리 컨테이너가 고객 인프라의 타 워크로드와 **격리**되고(필요시 Confidential Containers), 필요한 권한만 요청하는지 배포 문서에 명시.
- **이미지 최적화**: 이미지 크기·기동 시간 최적화 = 설치 경험·현장 리스크 감소.
- **GPU 요구사항 명세**: 필요한 GPU/메모리, **공유 가능 여부(HAMi)**, 유휴 최소화 방안 — 설치 전 고객 인프라팀과 합의.

### 5) 고객 인프라·서비스 안내 (Advisory)
- **AI/SW 지식으로 설명할 것**: K8s가 AI 워크로드 표준 기반(조직 66%가 컨테이너로 genAI), GPU 공유·사이징, 관측(OTel), 데이터 주권(온프렘의 가치).
- **안내 스크립트**: "우리 솔루션이 고객 인프라에 **어떻게 앉고(컨테이너/격리)**, **어떤 리소스(GPU·메모리)** 를 쓰고, **데이터가 어디 머무나(사내)**, **어떻게 감사되나**"를 한 장으로.

| 용어 | 한 줄 정의 | 왜 PM이 알아야 하나 | 고객/팀 대화에서 쓰는 법 |
|------|-----------|--------------------|------------------------|
| **MCP** | 에이전트↔도구/데이터 연결 표준 프로토콜 | 솔루션을 고객 시스템에 연동하는 방식 | "MCP로 귀사 시스템과 표준 연동됩니다" |
| **자체호스팅 추론(vLLM)** | 고객 인프라 안에서 모델을 직접 서빙 | 데이터 주권 요구 대응·원가 통제 | "모델을 귀사 서버에서 구동해 데이터가 외부로 안 나갑니다" |
| **Confidential Containers / 격리** | 실행 중 데이터까지 보호하는 컨테이너 격리 | 고객 보안심사 대응 | "타 워크로드와 격리되고 실행 중 데이터도 보호됩니다" |
| **ML 공급망 보안** | 모델·이미지 서명·스캔·SBOM으로 무결성 보장 | 배포 승인 조건 | "이미지 서명·취약점 스캔·SBOM을 제공합니다" |
| **거버넌스/감사로그** | 누가·무엇을·언제 했는지 추적·통제 | 규제·신뢰의 핵심 | "모든 작업이 감사로그로 추적되고 권한은 최소화됩니다" |
| **GPU 활용률/HAMi 공유** | 한 GPU를 여러 워크로드가 분할 사용 | 인프라 원가·견적 근거 | "GPU 공유로 유휴를 줄여 하드웨어 비용을 낮춥니다" |

## 🗣️ 고객 미팅에서 써먹을 한 줄 (Talking Points)
- "**데이터는 귀사 인프라를 벗어나지 않습니다** — 온프렘 Docker 배포 + 자체 추론."
- "이미지는 **서명·취약점 스캔·SBOM**을 제공해 보안 심사를 통과합니다."
- "모든 작업은 **감사로그**로 추적되고, 컨테이너는 **격리·최소권한**으로 배포됩니다."
- "GPU는 **공유·사이징**으로 유휴를 줄여 원가를 낮춥니다."
- "업계는 지금 성능이 아니라 **신뢰성·거버넌스**로 경쟁하며, 저희 솔루션은 그 기준에 맞춰 설계됐습니다."
